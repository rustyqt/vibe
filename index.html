Core Features

Everything included, nothing to configure

GitHub Pages gives you free, reliable hosting with a short list of powerful capabilities built in from the start.

🌐

Free Custom Domains

Point any domain you own (e.g. yourname.com) to your Pages site. HTTPS is provisioned automatically via Let's Encrypt.

🔒

Automatic HTTPS

Every site gets a free TLS certificate. No Certbot, no renewal scripts — GitHub manages the full lifecycle.

⚡

Global CDN

Content is served from GitHub's global edge network, so visitors worldwide get fast load times without any CDN setup on your part.

🔨

Jekyll Built-in

Push Markdown and let GitHub build your static site with Jekyll — themes, layouts, and Liquid templates included.

🔄

GitHub Actions CI/CD

Trigger any build pipeline on push. Use Actions to compile React, Vue, Next.js, Hugo, Astro — then deploy automatically.

📊

Branch & PR Previews

Deploy from any branch. With Actions you can publish preview deployments for every pull request to review changes before merging.

Quick Start

Live in under 5 minutes

Getting a site online requires just a few Git operations and one settings toggle. No terminal mastery needed.

1

Create a repository

Name it username.github.io for a user site, or any name for a project site (published at /repo-name).

2

Add your files

Push an index.html (or any static site output) to the repo. Works with hand-written HTML or any framework's build output.

3

Enable Pages in Settings

Go to Settings → Pages, choose a source branch (or GitHub Actions), and click Save.

4

Your site is live

Within seconds GitHub builds and deploys your site. Every subsequent push automatically republishes.

Automation

Deploy with GitHub Actions

Use a workflow file to build any framework and publish the output. The example deploys a Vite project on every push to main.

Swap the build step for npm run build:hugo, bundle exec jekyll build, or any other command — Pages doesn't care what generates the HTML.

.github/workflows/deploy.yml 
# Deploy a Vite app to GitHub Pages
name: Deploy
on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci && npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist
      - uses: actions/deploy-pages@v4
Limits & Pricing

What are the constraints?

GitHub Pages is free for public repos and included in all GitHub plans. Know the limits before you ship.

Limit Value Notes
Repository size 1 GB Recommended max; larger repos will trigger a warning.
Published site size 1 GB The actual HTML/CSS/JS served to visitors.
Monthly bandwidth 100 GB Soft limit — GitHub may contact high-traffic sites.
Build frequency 10 / hour Jekyll builds triggered directly (not via Actions).
Custom domains Unlimited One per Pages site; HTTPS via Let's Encrypt.
Price (public repos) Free Always free for open-source projects.
Price (private repos) Paid plans GitHub Pro, Team, or Enterprise required.
